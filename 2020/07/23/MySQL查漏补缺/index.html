MySQL查漏补缺

  • MySQL查询语句格式

  • select语句解析顺序

    1) FROM：对查询指定的表计算笛卡尔积；
    2) ON：按照join_condition过滤数据；
    3) JOIN：添加关联外部表数据；
    4) WHERE：按照where_condition过滤数据；
    5) GROUP BY：进行分组操作；
    6) HAVING：按照having_condition过滤数据；
    7) SELECT：选择指定的列；
    8) DISTINCT：指定列去重；
    9) ORDER BY：按照order_by_condition排序；
    10) LIMIT：取出指定记录的数量；（开始位置0开始，取数个数）

  • 创建表

  • 约束

    主键约束：它能够唯一确定一张表中的一个记录，也就是我们通过给某个字段添加约束，就可以使得该字段不重复且不为空。

    primary key 可以在字段后面添加，也可以在最后一起添加

    创建表之后想添加主键约束：

    alter table 表名 add primary key(id);

    alert table 表名 modify id int primary key;

创建表之后删除主键：alter table 表名 drop primary key;

联合主键约束：primary key（id，name）

两个主键值加起来不重复就可以

​ 自增约束：auto_increment 自动增长主键

​ 唯一约束：约束修饰字段的值不能重复

​ alert table 表名 add unique(键名);

​ 删除唯一约束：alert table 表名 drop index 键名；

​ 通过modify添加：alert table 表名 modify name varchar(20) unique;

  • 总结：

    1、可以建表的时候就添加约束

    2、可以使用alter 。。。add 。。。

    3、alter 。。。 modify 。。。

    4、删除 alter 。。。 drop 。。。

​ 非空约束：NOT NULL

​ 默认约束：当我们插入字段值的时候，如果没有传值就会使用默认值 default 值

​ 外键约束：涉及到两个表：父表、子表

​ foreign key(子表键名) references 父表(键名) 子表中不能添加主表中没有的数据值

  • 数据库的三大设计范式：

  • 1、数据表中的所有字段都是不可分割的原子值

    字段值还可以继续拆分的，就不满足第一范式

    比如：中国广东省深圳市龙城街道100号，就是可以继续拆分的

    范式，设计的越详细，对于某些操作了能更好，也不一定

  • 2、第二范式：

    必须满足第一范式的前提下，要求除主键外的每一列都必须·1完全依赖主键。如果要出现不完全依赖，只能发生在联合主键的情况下

    简称就是一个表中有两个主键，使得一些字段不完全依赖一个主键

  • 3、第三范式：

    必须先满足第二范式，除开主键列的其他列之间不能有传递依赖关系

事务的四大特征：

  • 原子性：事务时最小的单位，不可以再分割
  • 一致性：事务要求，同一事物中的sql语句，必须保证同时成功或者同时失败
  • 隔离性：事务1和事务2之间具有隔离性的
  • 持久性：事务一旦结束（commit，rollback），就不可以返回

事务开启：

  • 修改默认提交 set autocommit=0；
  • begin；
  • star transaction；

事务手动提交：

commit；

事务手动回滚：

rollback；

事务隔离性：

  • read uncommited 读未提交的
  • read commit 读已经重复的
  • repeatable reaad 可以重复读
  • serializable 串行性
